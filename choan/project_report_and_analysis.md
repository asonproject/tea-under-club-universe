# 📝 長安 (Choan) ウェブサイト プロジェクト報告書・分析資料

本資料では、今回構築した「長安」ウェブサイトの運用ガイド、将来的な発展性、ビジネスとしての市場価値、およびセキュリティ分析についてまとめます。

---

## 1. 📖 ページの使い方・運用ガイド

### 👥 閲覧者（顧客）の利用フロー
1.  **トップページ (`index.html`)**:
    *   🆕 新着情報で最新のイベントやメニューを確認。
    *   🍱「オードブル・仕出し弁当」、🍽️「ケータリング」の各ボタンから詳細ページへ遷移。
2.  **詳細ページ (`index-order.html`, `index-catering.html`)**:
    *   👀 具体的なメニュー内容、価格、写真を確認。
    *   **📞 注文・問い合わせ**: ページ下部の電話番号 (`050-3163-9877`) またはメールリンクから直接連絡。
    *   **🔙 戻る**: 画面下部の「トップページへ戻る」ボタンでホームに復帰。

---

### 💻 管理者（運営者）の運用タスク
1.  **🆕 新着情報の更新**:
    *   **方法**: 連携している **Googleスプレッドシート** に行を追加するだけで、自動的にウェブサイトの「新着情報」セクションに反映されます。HTMLを編集する必要はありません。
2.  **✏️ メニュー・価格の変更**:
    *   **方法**: 対応するHTMLファイル (`index-order.html` または `index-catering.html`) のテキスト部分を直接編集します。
3.  **🖼️ 画像の差し替え**:
    *   **方法**: 新しい画像を `./assets/` フォルダに保存し、ファイル名が同じであれば上書き、異なる場合はHTML内の `src` 属性を変更します。

---

## 2. 🚀 今後の発展・アップグレード提案

現状は「高速・軽量・セキュア」な静的サイトとして構築されています。ビジネスの成長に合わせて、以下の段階的なアップグレードを推奨します。

### フェーズ1：利便性の向上 (現状の延長)
*   **✅ お問い合わせフォームの設置 ✉️**:
    *   現在は電話とメールリンクのみですが、Googleフォームの埋め込みや、静的サイト向けフォームサービス（Formspreeなど）を導入することで、24時間受付可能な体制を作ります。
*   **✅ SNS連携の強化 📱**:
    *   Instagramのフィードを自動でサイト内に表示させ、ビジュアル面での訴求を強化します。

---

### フェーズ2：顧客体験の深化
*   **📅 Web予約・注文システムの導入**:
    *   電話での聞き間違いを防ぐため、メニューを選択して数量を入力できるカート機能や予約システムを追加します（リクルート「Airリザーブ」などの外部ツール連携が低コストでおすすめ）。
*   **💬 LINE公式アカウント連携**:
    *   「詳細はこちら」ボタンの一部をLINE友だち追加に誘導し、リピーター獲得につなげます。

### フェーズ3：ブランディング・本格運用
*   **🌐 「隠しページ」からの脱却**:
    *   現在設定されている `noindex`（検索除け）を解除し、SEO対策（地域名＋「ケータリング」「弁当」などでの検索上位表示）を本格的に行い、新規顧客の自然流入を狙います。
*   **🖥️ CMS化 (WordPress等)**:
    *   HTMLの直接編集が手間になった場合、管理画面から誰でも簡単に文字や画像を更新できるシステムへ移行します。

## 4. 🔒 セキュリティ SWOT分析

現状のシステム構成（静的HTMLサイト）におけるセキュリティ面を分析します。

| 項目                   | 内容                                                                                                                                                                                                                                                                                                                                                                                                          |
| :--------------------- | :------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------ |
| **💪 Strengths (強み)** | **🛡️ 攻撃対象が少ない**: <br>・データベースやサーバーサイドプログラム（PHPなど）を持たないため、SQLインジェクションやCMSの脆弱性を突いた攻撃を受けるリスクがほぼゼロです。<br>**🚫 個人情報を保持しない**: <br>・サイト上に顧客データを保存しないため、情報漏洩事故の心配がありません。<br>**🙈 隠蔽性**: <br>・`noindex` 設定により、無差別なクローラーのアクセスやそこからの攻撃スキャンを受けにくい状態です。 |

---

| 項目                       | 内容                                                                                                                                                                                                                                                                                                                                                                                                             |
| :------------------------- | :--------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **😓 Weaknesses (弱み)**    | **📧 メールアドレスの露出**: <br>・`info@...` をテキストで掲載しているため、スパムボットに収集されやすく、迷惑メールが増える可能性があります。<br>**🔓 アクセス制御の限界**: <br>・`noindex` はあくまで「検索結果に出ない」だけであり、URLを知っている人は誰でも閲覧可能です。会員限定等の厳密な閲覧制限機能はありません。                                                                                         |
| **🌟 Opportunities (機会)** | **🔐 SSL/HTTPS化**: <br>・通信の暗号化は必須です（現在はホスティング環境に依存しますが、無料のLet's Encryptなどで容易に実装可能）。<br>**🛡️ WAF (Web Application Firewall) 導入**: <br>・サーバーレベルでWAFを導入することで、DDoS攻撃などの脅威をさらに軽減できます。<br>**📝 スパム対策フォームへの移行**: <br>・メールアドレスの直接掲載をやめ、Googleフォーム等を導入することでスパムメールを劇的に減らせます。 |

---

| 項目                 | 内容                                                                                                                                                                                                                                                                                                                                                                                                                                                         |
| :------------------- | :----------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------------- |
| **⚠️ Threats (脅威)** | **🎣 なりすまし**: <br>・サイトのデザインをコピーしたフィッシングサイトが作られるリスク（ドメインの管理と、SPF/DKIM等のメールセキュリティ設定で対抗）。<br>**💥 ホスティングサービスの障害**: <br>・サイトデータ自体は安全でも、利用しているサーバー（レンタルサーバーやGitHub Pages等）がダウンすると閲覧不能になります。<br>**🔑 FTP/管理パスワードの流出**: <br>・更新者のPCがウイルス感染し、サーバーへのアップロード権限を盗まれることが最大のリスクです。 |
